# 游戏数值系统设计文档

## 1. 系统概述

本文档定义了游戏中的核心数值系统，包括角色属性、数值范围、影响机制等内容。该系统通过各种数值维度刻画角色的性格、道德底线和人物关系。

## 2. 角色体系

### 2.1 摄影师（用户操控角色）

#### 属性定义

| 属性名称 | 范围 | 初始值 | 说明 |
|---------|------|-------|------|
| 堕落度 | 0 ~ 100 | 0 | 道德底线的衡量，数值越高越容易接受非法/不道德的任务 |

#### 属性详解

**堕落度 (0 ~ 100)**
- **定义**：摄影师的道德底线指数，反映其在追求利益过程中的原则性
- **行为表现**：
  - **0-20 (清廉)**：拒绝不道德的任务，仅接受合法的商业拍摄
  - **21-40 (妥协)**：开始接受边界工作，如半裸拍摄等
  - **41-60 (沉沦)**：接受大尺度拍摄，不再关心模特感受
  - **61-80 (堕落)**：为了利益不顾一切，可能进行非法拍摄
  - **81-100 (深渊)**：完全丧失底线，可进行极端和非法活动
- **获得方式**：
  - 接受不道德任务时 +5~15
  - 强迫模特进行超出协议的拍摄时 +10~20
  - 选择金钱而非道德时 +5~10
- **后果**：
  - 解锁不同难度的任务类型
  - 影响与模特的互动选项
  - 影响故事结局

### 2.2 女模特

#### 属性定义

| 属性名称 | 范围 | 初始值 | 说明 |
|---------|------|-------|------|
| 好感度 | -100 ~ 100 | 0 | 模特对摄影师的好感程度和关系亲密度 |
| 堕落度 | 0 ~ 100 | 0 | 模特的道德底线指数 |

#### 属性详解

**好感度 (-100 ~ 100)**

- **定义**：模特对摄影师的情感态度，分为厌恶、中立和喜欢三个阶段
- **价值区间**：
  - **[-100, -50)**：深度厌恶
    - -100：死仇，模特会拒绝所有合作
    - -99 ~ -50：极度厌恶，可能采取极端行为
  - **[-50, -1)**：厌恶/不满
    - -49 ~ -20：明显厌恶，需要高额报酬才会合作
    - -19 ~ -1：冷淡，但基本合作
  - **[0, 0]**：中立
    - 尚未建立深层关系或完全陌生
    - 无特殊感受，可接受基本商业合作
  - **(0, 50]**：轻微喜欢
    - 1 ~ 20：无恶意的陌生人，产生良好印象
    - 21 ~ 50：初步好感，愿意主动合作
  - **(50, 100]**：深度好感
    - 51 ~ 80：好感，信任摄影师
    - 81 ~ 100：深爱，可能为摄影师做任何事

- **变化因素**：
  - 摄影师的堕落度越高，好感度容易下降
  - 摄影师的行为选择直接影响变化
  - 提供高报酬可在短期内增加好感度
  - 不尊重模特会快速降低好感度
  - 强迫拍摄会大幅降低好感度

**堕落度 (0 ~ 100)**

- **定义**：模特的道德和心理防线，反映其对自身底线的放弃程度
- **行为表现**：
  - **0-20 (清纯)**：对自身身体有清晰的底线，拒绝不合理要求
  - **21-40 (松动)**：开始考虑金钱诱惑，底线开始动摇
  - **41-60 (妥协)**：为了金钱接受一些不规范拍摄
  - **61-80 (沉沦)**：把身体当作商品，可以接受大尺度内容
  - **81-100 (堕落)**：完全丧失自我认知，沦为工具，甚至对自己的遭遇习以为常
- **获得方式**：
  - 被强迫进行超出协议的拍摄时 +10~30
  - 经历多次不尊重的对待时 +5~10
  - 摄影师堕落度高时，每次互动的堕落度增幅加倍
  - 接受高额报酬时 +3~8
- **影响**：
  - 解锁不同的剧情分支
  - 影响模特的心理状态和行为
  - 影响最终故事结局

## 3. 交互系统

### 3.1 任务系统

#### 任务类型分类

| 任务类型 | 所需最低堕落度 | 风险等级 | 报酬倍数 |
|---------|---------------|---------|---------|
| 商业拍摄 | 0 | 低 | 1x |
| 半裸写真 | 20 | 中 | 2x |
| 限制级拍摄 | 40 | 中 | 3x |
| 非法拍摄 | 60 | 高 | 5x |
| 极端拍摄 | 80 | 极高 | 10x |

### 3.2 交互选项

根据摄影师和模特的属性值，系统会解锁不同的对话和行为选项。

**示例交互矩阵**：
- 摄影师堕落度 < 20，模特好感度 > 50：解锁"温柔建议"选项
- 摄影师堕落度 > 60，模特好感度 < 0：解锁"强迫"选项
- 摄影师堕落度 > 80，模特堕落度 > 80：解锁"极端内容"路线

## 4. 数值变化机制

### 4.1 摄影师堕落度变化

```
基础变化 = 任务风险等级 × 选择权重

选择权重：
- 尊重模特意愿：×0.5（甚至可能 -5）
- 中立态度：×1.0
- 利用模特：×1.5
- 强迫模特：×2.0
```

### 4.2 模特好感度变化

```
基础变化 = 摄影师行为评分 - 摄影师堕落度影响

摄影师行为评分：
- 尊重边界：+10
- 公平交易：+5
- 强行扩展边界：-20
- 强迫拍摄：-50
- 骚扰/侵犯：-100
```

### 4.3 模特堕落度变化

```
基础变化 = 创伤程度 × (1 + 摄影师堕落度/100)

创伤程度分析：
- 被强迫一次：+10~20
- 被多次强迫：+15~25
- 被精神操纵：+20~30
- 被侵犯：+50+
```

## 5. 故事分支和结局

### 5.1 结局条件

| 结局类型 | 摄影师堕落度 | 模特好感度 | 模特堕落度 |
|---------|-----------|---------|---------|
| 真实结局（好） | 0-30 | 50+ | 0-30 |
| 平衡结局 | 20-50 | -20~50 | 20-60 |
| 悲剧结局（坏） | 50+ | -100~ -30 | 60+ |
| 黑暗结局 | 80+ | -100 | 100 |

### 5.2 结局描述

- **真实结局（好）**：摄影师和模特建立了健康的职业和个人关系，互相尊重
- **平衡结局**：两人都在灰色地带中徘徊，各有得失
- **悲剧结局（坏）**：模特心理崩溃，摄影师陷入深度堕落
- **黑暗结局**：双方都完全陷入深渊，无法自拔

## 6. UI 显示建议

### 6.1 属性条形图

- 堕落度用**红色**表示（越红越堕落）
- 好感度用**蓝色和红色混合**表示（正值蓝色，负值红色）

### 6.2 状态标签

根据数值范围显示对应的状态标签：
- 堕落度："清廉"/"妥协"/"沉沦"/"堕落"/"深渊"
- 好感度："死仇"/"厌恶"/"冷漠"/"中立"/"好感"/"信任"/"深爱"

## 7. 平衡性考虑

### 7.1 设计原则

1. **选择权重**：游戏应该让玩家能够在不同的道路上进行，而不是强制某一条路线
2. **反馈机制**：数值变化应该清晰可见，让玩家理解自己的行为后果
3. **重玩价值**：不同的属性组合应该解锁不同的故事内容

### 7.2 难度调整

- 新手模式：模特初始好感度 +20，堕落度变化速度减半
- 标准模式：按设计文档运行
- 困难模式：模特初始好感度 -10，堕落度变化速度加倍

## 8. 扩展考虑

### 8.1 未来可能的新属性

- **声誉值**：摄影师在业界的名声
- **心理健康**：模特的心理状态详细度量
- **经济状况**：双方的财务状况
- **社会压力**：外界对两人的影响

### 8.2 动态任务生成

根据当前属性值动态生成任务：
- 高堕落度时出现"黑暗任务"
- 高好感度时出现"亲密互动"
- 高模特堕落度时出现"极限任务"

## 9. 技术实现建议

### 9.1 数据结构

```javascript
// 摄影师数据结构
photographer = {
  corruptionLevel: 0,  // 0-100
  reputation: 0,
  money: 1000,
  history: []  // 记录所有决定
}

// 模特数据结构
model = {
  affection: 0,        // -100~100
  corruptionLevel: 0,  // 0-100
  psychologicalState: 'normal',
  traumaLevel: 0,
  money: 0,
  history: []
}
```

### 9.2 事件系统

创建事件驱动的系统，监听属性变化：
- 当堕落度达到某个阈值时触发事件
- 当好感度变化时更新可用选项
- 当两者属性特定组合时触发特殊事件

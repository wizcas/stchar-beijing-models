# 状态栏 Schema 快速参考

## 📊 主要层级结构

```
状态栏
├─ 世界 (world)
│  ├─ 时间 (ISO8601+时区)
│  ├─ 地点
│  ├─ 天气
│  └─ 出场女模 (JSON 数组)
├─ user
│  ├─ 想法
│  ├─ 资金 (数字)
│  ├─ 拍摄任务 (完整 7 个字段)
│  └─ 器材 (5 个数组字段)
├─ 女模 {名字}
│  ├─ 想法
│  ├─ 关系 (堕落度/堕落度描述/好感度/好感度描述)
│  ├─ 外型 (9 个字段)
│  ├─ 职业 (5 个字段)
│  └─ 性爱 (性癖/性经验次数/动情程度/[生理字段])
├─ 推进选项 (4 个数组，2 积极 + 2 消极)
└─ 历史.女模 (字符串数组)
```

---

## 🔴 强制规则：关联字段同步

当更新这些字段时，**必须**同时更新关联字段：

| 更新字段 | 必须同时更新 | 条件 |
|---------|-----------|------|
| `user.堕落度` | `user.堕落度描述` | 总是 |
| `女模.关系.好感度` | `女模.关系.好感度描述` | 总是 |
| `女模.关系.堕落度` | `女模.关系.堕落度描述` | 总是 |
| `女模.性爱.动情程度` | `乳房`、`小穴`、`肛门` | ≥10 时 |

---

## ✅ 必须项检查

- [ ] 新女模：包含完整结构（关系、外型、职业、性爱）
- [ ] 性爱初始值：性癖=`[]`，动情程度=`0`
- [ ] 生理字段：仅在动情程度≥10 时生成
- [ ] 任务字段：7 个必填（发布日期/状态/模特/目标/期限/报酬/特殊要求）
- [ ] 出场女模：JSON 数组 `["名1", "名2"]`

---

## 禁止项

- ❌ YAML 格式（`-` 开头列表）
- ❌ 创造新字段
- ❌ 只更新数值，遗漏描述字段
- ❌ 刻板化陈词滥调（描述需贴切情境）

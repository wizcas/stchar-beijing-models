# Plot-Log 核心规则速查

## 🔴 三大绝对规则

### 1. 使用 JSON 格式，禁止 YAML
```
❌ 禁止: 状态栏.出场女模: - 六花
✅ 正确: {"状态栏.出场女模": ["六花"]}
```

### 2. 动作必须单一且合并
- `"记下"` — 只能出现 1 次（所有赋值合并）
- `"调整"` — 只能出现 1 次（所有调整合并）
- `"追加"` — 只能出现 1 次（所有追加合并）
- **禁止**：同一动作出现多次

### 3. JSON 必须合法可解析
- 所有字符串用双引号 `""`
- 数字不使用引号（`5000` 不是 `"5000"`）
- 调整值为字符串（`"+10"` 不是 `+10`）
- 逗号位置正确（最后一项无逗号）

### 4. 字段名必须是简体中文（🔴 严禁繁体）
- ✅ 正确：`"状态栏"`、`"出场女模"`、`"关系"`、`"性爱"`
- ✅ 正确：`"堕落度"、`"好感度"`、`"动情程度"`
- ❌ 严禁：`"狀態欄"`、`"關係"`、`"性愛"`、`"墮落度"`、`"動情程度"`
- ❌ 严禁：混合使用简体和繁体

**验证方法**：所有中文字段名必须能在简体中文输入法下直接输入

---

## 📋 快速检查清单

**格式检查**：
- [ ] 使用 JSON `[]` 格式，不用 YAML `-`
- [ ] `"记下"` 只出现 1 次
- [ ] `"调整"` 只出现 1 次
- [ ] `"追加"` 只出现 1 次
- [ ] JSON 能被 `JSON.parse()` 解析
- [ ] ⚠️ **所有中文字段名是简体中文，严禁繁体**

**内容检查**：
- [ ] 出场女模与场景一致
- [ ] 任务包含全部 7 个字段（含发布日期）
- [ ] 推进选项恰好 4 个（2 积极 + 2 消极）
- [ ] 更新好感度 → 同时更新好感度描述
- [ ] 更新堕落度 → 同时更新堕落度描述
- [ ] 动情程度≥10 → 同时更新乳房、小穴、肛门
